dkim_signalg_t

[back to index]

DESCRIPTION
Purpose A dkim_signalg_t is a value from an enumerated list of the possible message signing algorithms available. It can be optained via dkim_sig_getsignalg() after verification of a message. It need not be passed to the dkim_sign() or dkim_verify() functions as they will determine the message signing algorithm for signing based on the type of key and hash algorithm provided and for verification based on the signature on an arriving message.
Possible Values
  • DKIM_SIGN_RSASHA1 -- use an RSA-signed SHA1 hash of the message's headers and body
  • DKIM_SIGN_RSASHA256 -- use an RSA-signed SHA256 hash of the message's headers and body
  • DKIM_SIGN_ED25519SHA256 -- use an Ed25519-signed SHA256 hash of the message's headers and body
NOTES
  • The DKIM_SIGN_RSASHA256 and DKIM_SIGN_ED25519SHA256 message signing algorithms are only available if the cryptography library available at compile-time supported the respective SHA-256 and Ed25519 algorithms.
